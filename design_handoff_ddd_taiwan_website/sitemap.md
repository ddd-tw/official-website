# DDD Taiwan 官網架構 Site Map

> 本文件描述 DDD Taiwan 官方網站的頁面結構、內容模組與自動化流程規劃。

---

## 1. 參與活動 — 同步 / 即時互動

本區塊提供社群成員即時參與的活動資訊，強調人與人之間的實體或線上互動。

### 1.1 Meetup
- 定期舉辦的技術聚會
- 提供活動報名、場地資訊、議程預告
- 活動結束後補充錄影回放連結

### 1.2 Conference
- 已有的網站 https://conference.ddd-tw.com/

### 1.3 Book Club
- 定期讀書會資訊
- 目前閱讀書目與進度
- 加入方式與討論管道說明

---

## 2. 知識內容庫 — 非同步 / 自學資源

本區塊作為 DDD 領域知識的集散地，供社群成員自主學習與參考。

### 2.1 Books
- DDD 相關書單推薦
- 書籍摘要、評論與閱讀心得

### 2.2 Papers
- 領域相關論文與技術文章
- **上架流程（Agent 輔助）**：
  1. Agent 自動從網路抓取相關文章
  2. 聯繫原作者取得授權同意
  3. 社群 Review 審核
  4. 通過後正式發布至網站

### 2.3 Video
- Meetup、Conference 錄影回放
- 外部推薦影片彙整
- 依主題 / 難度分類瀏覽

### 2.4 GitHub Repositories
- DDD 相關開源專案收錄
- 社群成員貢獻的範例程式碼
- 外部優質 Repository 推薦清單

---

## 3. 社群與人 — 連結與認同

本區塊引導訪客加入社群各平台，建立歸屬感與橫向連結。

| 平台 | 說明 | 狀態 |
|------|------|------|
| Facebook | 粉絲專頁 / 社團，日常貼文與活動公告 | ✅ 已有 |
| Line | 即時通訊群組，快速交流與問答 | ✅ 已有 |
| Twitter / Instagram | 短文分享與視覺內容 | 🔲 待確認 |

---

## 4. 治理與政策 — 規範層

本區塊確保社群運作透明，保障成員權益與互動品質。

### 4.1 Privacy Policy（隱私權政策）
- 個人資料蒐集、使用與保護說明
- 符合相關法規（如 GDPR、個人資料保護法）

### 4.2 Code of Conduct（社群行為準則）
- 社群互動規範與期望行為
- 違規處理流程
- 申訴與聯絡管道
- 講師不能攻擊別人

## 自動化流程（Agent）

> 以下功能透過 Agent 自動執行，降低人工維護成本，並解決帳號集中管理的瓶頸。

| # | 自動化項目 | 說明 |
|---|-----------|------|
| 1 | **Meetup 活動同步至 Calendar** | 透過 API 自動將 Meetup 活動連結新增至公開行事曆 |
| 2 | **自動上架 Meetup 活動** | 分散式上架機制，減少僅單一帳號可操作的限制 |
| 3 | **錄影通知信自動發送** | 若活動承諾錄影，於錄影完成後自動寄送影片連結給參與者 |
| 4 | **錄影自動上傳 YouTube** | Meetup 錄影完成後，自動發布至 DDD Taiwan YouTube 頻道 |
